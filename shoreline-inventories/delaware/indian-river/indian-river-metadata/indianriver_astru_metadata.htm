Description:
Abstract:
Shoreline Situation Reports (SSR) report conditions and status of shore lands. The reports are intended to assist planners, managers, and regulators in decisions pertaining to management of coastal areas and natural resources therein. Data collected describes conditions in the immediate riparian zone, the bank, and along the shore.
Purpose:
To inventory the Indian River, Delaware tidal shoreline conditions.
Supplemental_Information:
Indriv_astru (Indian River access structures) is a point shapefile with locations of docks, ramps (both public and private), and boathouses. The attributes of this dataset is explained in the Entity and Attribute Information section below.

Positional_Accuracy:
Horizontal_Positional_Accuracy:
Horizontal_Positional_Accuracy_Report:
Land use was recorded with a GPS unit as the boat moved along the shoreline. Data was transferred from the GPS boat track to an existing shoreline coverage by projecting data to the shoreline at a 90 degree angle from the boat track. Rectified digital orthophoto quads were used as a backround coverage to aid in data positioning. Land use features were placed along the shore so changes in land use were coincident with changes illustrated in the imagery. Positional accuracy for data that has been corrected with imagery is 2 meters (equivalent to image resolution). Accuracy was determined by comparing digitally processed data locations to onsite GPS ground surveys collected at random.

Process_Step:
Process_Description:
Data was collected by a two-person field crew (one data collector and one boat operator/data collector) using two hand-held GeoExplorer GPS units, while navigating along the shoreline. A data dictionary designed for the inventory was installed on each GPS unit. The data collector records natural features including land use, bank height, bank cover, presence of marsh, beach, and Phragmites Australis, tree fringe, and shoreline stability. The boat operator collects data on the hardened structures: riprap, bulkheads, docks, boat ramps, etc. Point features (docks, boathouses, ramps) were surveyed with a 10 second observation recorded at 1 reading/second. Linear features were surveyed kinematically at a rate of one observation ervery 3 seconds. Data collection began 6/26/2006 and ended 6/28/2006.
Process_Date: 2006
Process_Step:
Process_Description:
Data from the GPS units were postprocessed on a PC with Trimble Pathfinder Office software. Postprocessing included differential correction using data collected at various CORS (continuously operating reference stations) sites. Corrected data is converted into shapefiles for further processing.
Process_Date: 2006
Process_Step:
Process_Description:
ArcMap 9.1 software on a PC was used to attribute the base shoreline. Boat-track points and arcs, DOQ imagery, and the base shoreline were displayed in ArcMap. Shoreline arcs were split using the edit tool and the spatial adjustment attribute transfer tool was used to move the attributes from the boat track to the shoreline. Arcs were split on and points were moved to locations on the shoreline that were perpendicular to the original locations on the boat-track. Geodatabase features were visually checked for attribute accuracy between coded shoreline and boat-track.
Process_Date: 2006
Process_Step:
Process_Description:
Twelve 1:12,000 scale plates were created for Indian River. Each plate shows 3 maps: riparian land use, bank and buffer conditions, and shoreline features. DOQQ imagery is displayed in the background to provide users with additional information. Maps are created using ArcMap software.
